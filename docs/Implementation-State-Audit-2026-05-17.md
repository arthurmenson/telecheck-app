## 4. Pilot-launch pathway

The Master Completion Plan v1.0 Phase B "Clinical Care" track exit gate requires implementation-state milestones across 3 axes:

### Axis A — Pilot-required slices implementable

The 2026-05-15 conceptual audit (since absorbed into the per-SI severity-tagging used in SI-011/012/014) flagged 3 pilot-required slices as not-yet-implementable:

1. **med-interaction (SKELETON)** — `crit:SI-012 ratification`. Without it, the I-007 interaction-engine platform-floor cannot be enforced. ETA-to-implementable: same day as SI-012 ratifies. ETA-to-feature-complete: ~2-3 sprints after ratification (2000-3000 LOC).
2. **forms-intake publish-gate IMPL** — `crit:SI-011 ratification + four IMPL-readiness gates (SI-010 + SI-008 + MarketingCopy CDM + I-030 detection rules)`. Without these, the production-deploy gate is sentinel-bypassed. ETA-to-implementable: same calendar window as SI-011 + its four IMPL gates ratify. ETA-to-feature-complete: ~1-2 sprints after ratification (600-1000 LOC).
3. **ai-service Mode 1 patient surface for Telecheck-Ghana** — `crit:SI-014 ratification (option A/B/C ships classifier) OR SI-014 Option D ratification (patient-access gate + SI-014.1 successor)`. Without these, the regex-stub safety surface cannot meet I-019 for Telecheck-Ghana's expected Twi-language mix. ETA-to-implementable: depends on ADR-030 option choice (Option A: 4-6 weeks; Option B: 6-12 months; Option C: 4-6 weeks; Option D: same day with patient-access gate landing).

### Axis B — Platform invariants verifiable end-to-end

Per `docs/BUILD_VS_SPEC_TRACEABILITY_MATRIX.md` r6 §1: 11 invariants COMPREHENSIVE (I-003/009/012/014/016/019/023/025/026/027 + I-024 break-glass partial). I-029/030/031 (research data) BLOCKED at `inactive` per ADR-028 v0.5 — not a pilot-launch dependency.
