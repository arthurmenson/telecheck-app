## Append-only + Tier 0/1/2 invariants

### forms_template_l3_edit_log

**Tier 0 — Fully append-only from INSERT** (every column immutable post-INSERT; no UPDATE path; no DELETE path):
- All 13 columns are evidence artifacts; the row IS the change-data-capture
- Mirrors I-003 audit_records discipline + SI-015 MarketingCopyGovernanceEvidence pattern + SI-016 ai_workflow_handler_audit_log pattern

### forms_template_l3_approval

**Tier 0 — Fully append-only from INSERT** (every column immutable post-INSERT; no UPDATE path; no DELETE path):
- All 11 columns are evidence artifacts; the row IS the approval record
- Mirrors the audit_log discipline

### tenant_clinical_lead_assignments

**Tier 0 — Identity binding immutable from INSERT** (FROZEN at row creation; never mutable):
- `assignment_id`, `tenant_id`, `account_id`, `assigned_at`, `assigner_account_id`, `assigner_role_at_assignment`, `created_at` — assignment identity binding cannot be retroactively changed

**Tier 1 — Status-transition allow-list** (status column + revocation columns remain mutable, but ONLY through guarded `revoke_tenant_clinical_lead` SECURITY DEFINER procedure):
- `active → revoked` (atomically sets revoked_at + revoker_account_id + revoker_role_at_revocation + revocation_reason via the procedure)
- All other transitions FORBIDDEN; `revoked` is terminal (re-assignment creates a NEW row, not an UPDATE)

## Repository layer enforcement + helpers (pseudo-SQL)

```sql
-- is_active_edit_log: transitive supersession resolver (per SI-011 source line 58).
-- Returns TRUE if the edit_log_id is NOT transitively superseded by any later row.
-- Used by the publish-time active-row predicate (per SI-011 source line 69).
CREATE OR REPLACE FUNCTION is_active_edit_log(
  p_tenant_id   TEXT,
  p_edit_log_id TEXT
) RETURNS BOOLEAN
LANGUAGE plpgsql STABLE
SET search_path = pg_catalog, pg_temp
AS $$
DECLARE
  v_superseding_row_count INTEGER;
BEGIN
  -- A row is INACTIVE iff there exists a later row that supersedes it
  SELECT COUNT(*) INTO v_superseding_row_count
   FROM public.forms_template_l3_edit_log e
   WHERE e.tenant_id = p_tenant_id
     AND e.supersedes_edit_log_id = p_edit_log_id;
  RETURN v_superseding_row_count = 0;
END;
$$;

-- forms_template_l3_edit_log: BEFORE INSERT trigger forces evidence-grade timestamps
-- to DB time; computes new_value_hash from caller-supplied new_value via DB function;
-- enforces edit_type-specific column population requirements.
CREATE OR REPLACE FUNCTION forms_template_l3_edit_log_insert_guard()
RETURNS TRIGGER AS $$
DECLARE
  v_superseded_changed_path_set JSONB;
BEGIN
  NEW.edited_at := NOW();
  NEW.created_at := NOW();
  -- If this is a corrective_supersession, force superseded_at to NOW()
  IF NEW.edit_type = 'corrective_supersession' THEN
    NEW.superseded_at := NOW();
    -- Same-editor constraint: superseding editor must match this row's editor
    IF NEW.superseding_editor_account_id IS DISTINCT FROM NEW.editor_account_id THEN
      RAISE EXCEPTION 'forms_template_l3_edit_log: corrective_supersession requires superseding_editor_account_id = editor_account_id (single-author corrective edits)';
    END IF;
    -- Supersession coverage CHECK (per SI-011 source line 79): the corrective row's
    -- changed_path_set MUST cover every path in the superseded row's changed_path_set
    -- (set inclusion). Paths the corrective row does NOT cover would leave the superseded
    -- row's value live without a fresh approval — categorically forbidden.
    SELECT changed_path_set INTO v_superseded_changed_path_set
     FROM public.forms_template_l3_edit_log
     WHERE tenant_id = NEW.tenant_id AND edit_log_id = NEW.supersedes_edit_log_id;
    IF NOT FOUND THEN
      RAISE EXCEPTION 'forms_template_l3_edit_log: corrective_supersession references nonexistent supersedes_edit_log_id=%', NEW.supersedes_edit_log_id;
    END IF;
    -- Set-inclusion check: every path in v_superseded_changed_path_set MUST appear in NEW.changed_path_set
    IF EXISTS (
      SELECT 1 FROM jsonb_array_elements_text(v_superseded_changed_path_set) AS p
       WHERE p.value NOT IN (SELECT jsonb_array_elements_text(NEW.changed_path_set))
    ) THEN
      RAISE EXCEPTION 'forms_template_l3_edit_log: corrective_supersession changed_path_set must cover every path in superseded row''s changed_path_set (set inclusion required)';
    END IF;
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER forms_template_l3_edit_log_insert_lifecycle_guard
  BEFORE INSERT ON forms_template_l3_edit_log
  FOR EACH ROW
  EXECUTE FUNCTION forms_template_l3_edit_log_insert_guard();

-- forms_template_l3_edit_log: full append-only enforcement (UPDATE + DELETE rejected)

BEGIN

-- forms_template_l3_approval: BEFORE INSERT trigger forces approved_at + created_at
-- to DB time; copies approved_path_set + approved_value_fingerprint_map from the
-- referenced edit_log row at INSERT (deep-equal binding; defense-in-depth against
-- caller-supplied drift).
CREATE OR REPLACE FUNCTION forms_template_l3_approval_insert_guard()
RETURNS TRIGGER AS $$
DECLARE
  v_edit_log_row RECORD;
BEGIN
  NEW.approved_at := NOW();
  NEW.created_at := NOW();
  -- Load referenced edit_log row for binding validation + denormalization
  SELECT tenant_id, template_id, draft_revision_id, editor_account_id,
         changed_path_set, value_fingerprint_map
    INTO v_edit_log_row
   FROM forms_template_l3_edit_log
   WHERE tenant_id = NEW.tenant_id AND edit_log_id = NEW.edit_log_id;
  IF NOT FOUND THEN
    RAISE EXCEPTION 'forms_template_l3_approval: edit_log_id=% not found in tenant=%', NEW.edit_log_id, NEW.tenant_id;
  END IF;
  -- Cross-validation: caller-supplied denormalized fields MUST match the edit_log row
  IF NEW.template_id IS DISTINCT FROM v_edit_log_row.template_id
     OR NEW.draft_revision_id IS DISTINCT FROM v_edit_log_row.draft_revision_id
     OR NEW.editor_account_id IS DISTINCT FROM v_edit_log_row.editor_account_id THEN
    RAISE EXCEPTION 'forms_template_l3_approval: caller-supplied (template_id, draft_revision_id, editor_account_id) must match edit_log row';
  END IF;
  -- Force approved_path_set + approved_value_fingerprint_map to deep-equal copies of edit_log values
  -- (caller-supplied values silently overridden; eliminates caller-arbitrary-binding attack)
  NEW.approved_path_set := v_edit_log_row.changed_path_set;
  NEW.approved_value_fingerprint_map := v_edit_log_row.value_fingerprint_map;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

  BEFORE INSERT ON forms_template_l3_approval

RETURNS TRIGGER AS $$
BEGIN

  NEW.updated_at := NOW();

    RAISE EXCEPTION 'tenant_clinical_lead_assignments(% / %): direct INSERT cannot pre-seed revocation columns', NEW.tenant_id, NEW.assignment_id;
  END IF;

$$ LANGUAGE plpgsql;

-- tenant_clinical_lead_assignments: Tier 0 identity immutability (BEFORE UPDATE trigger):
CREATE OR REPLACE FUNCTION tenant_clinical_lead_assignments_tier0_immutability()
RETURNS TRIGGER AS $$
BEGIN
  IF NEW.assignment_id IS DISTINCT FROM OLD.assignment_id
     OR NEW.tenant_id IS DISTINCT FROM OLD.tenant_id
     OR NEW.account_id IS DISTINCT FROM OLD.account_id
     OR NEW.assigned_at IS DISTINCT FROM OLD.assigned_at
     OR NEW.assigner_account_id IS DISTINCT FROM OLD.assigner_account_id
     OR NEW.assigner_role_at_assignment IS DISTINCT FROM OLD.assigner_role_at_assignment
     OR NEW.created_at IS DISTINCT FROM OLD.created_at THEN
    RAISE EXCEPTION 'tenant_clinical_lead_assignments(% / %): Tier 0 identity immutability violated', NEW.tenant_id, NEW.assignment_id;
  END IF;
  -- Status-transition allow-list (only active → revoked permitted; revoked is terminal)
  IF OLD.status = NEW.status THEN
    -- Same-state UPDATE: no rewrites of any column permitted (Tier 0 + revocation columns also frozen)
    IF NEW.revoked_at IS DISTINCT FROM OLD.revoked_at
       OR NEW.revoker_account_id IS DISTINCT FROM OLD.revoker_account_id
       OR NEW.revoker_role_at_revocation IS DISTINCT FROM OLD.revoker_role_at_revocation
       OR NEW.revocation_reason IS DISTINCT FROM OLD.revocation_reason THEN
      RAISE EXCEPTION 'tenant_clinical_lead_assignments(% / %): same-state UPDATE (status=%) cannot rewrite revocation columns', NEW.tenant_id, NEW.assignment_id, OLD.status;
    END IF;
    NEW.updated_at := NOW();
    RETURN NEW;
  END IF;
  IF OLD.status = 'active' AND NEW.status = 'revoked' THEN
    NEW.revoked_at := NOW();
    NEW.updated_at := NOW();
    IF NEW.revoker_account_id IS NULL OR NEW.revoker_role_at_revocation IS NULL OR NEW.revocation_reason IS NULL THEN
      RAISE EXCEPTION 'tenant_clinical_lead_assignments(% / %): active → revoked requires revoker_account_id + revoker_role_at_revocation + revocation_reason', NEW.tenant_id, NEW.assignment_id;
    END IF;
    -- Cross-validate revoker_role_at_revocation against current accounts.role for revoker_account_id
    -- (mirror SI-015 R3 closure approver-role cross-validation)
    PERFORM 1 FROM accounts a
      WHERE a.tenant_id = NEW.tenant_id AND a.account_id = NEW.revoker_account_id AND a.role = NEW.revoker_role_at_revocation;
    IF NOT FOUND THEN
      RAISE EXCEPTION 'tenant_clinical_lead_assignments(% / %): revoker_role_at_revocation=% does not match current accounts.role for revoker_account_id=% — role snapshot must reflect provably-current authorization state', NEW.tenant_id, NEW.assignment_id, NEW.revoker_role_at_revocation, NEW.revoker_account_id;
    END IF;
    RETURN NEW;
  END IF;
  RAISE EXCEPTION 'tenant_clinical_lead_assignments(% / %): forbidden status transition % → %; allow-list: active → revoked only (revoked is terminal; re-assignment is a NEW row)', NEW.tenant_id, NEW.assignment_id, OLD.status, NEW.status;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER tenant_clinical_lead_assignments_state_machine_guard
  BEFORE UPDATE ON tenant_clinical_lead_assignments
  FOR EACH ROW
  EXECUTE FUNCTION tenant_clinical_lead_assignments_tier0_immutability();

RETURNS TRIGGER AS $$

END;
$$ LANGUAGE plpgsql;
```

```sql
-- assign_tenant_clinical_lead: INSERT new active assignment + atomic audit emission
CREATE OR REPLACE PROCEDURE assign_tenant_clinical_lead(

LANGUAGE plpgsql SECURITY DEFINER
```

## Publish-path L3 dual-control gate (informational; implementation lands at templateService.publishVersion)

The publish-time L3 dual-control gate (per SI-011 source §SI-011a step 3 + step 4) consumes the SI-011.1a row shapes:

1. **Active-row predicate**: filter `forms_template_l3_edit_log` rows where `is_active_edit_log(tenant_id, edit_log_id) = TRUE` (transitive supersession resolver)
2. **Per-edit-log paired-approval check**: for each active edit-log row, require exactly one `forms_template_l3_approval` row bound 1:1 via the composite FK + UNIQUE
3. **Publish-time revalidation**: re-check `approval.approved_path_set = edit_log.changed_path_set` AND `approval.approved_value_fingerprint_map = edit_log.value_fingerprint_map` AT PUBLISH TIME (defense-in-depth on top of INSERT-time CHECKs)
4. **State-validating gate**: every live `eligibility_logic` leaf MUST trace to an active approved edit (per SI-011 source §SI-011a step 4)
5. **Approver role validation**: `approval.approver_role_at_approval = 'tenant_clinical_lead'` AND the approver MUST currently have an active assignment in `tenant_clinical_lead_assignments`

Audit emission paths consumed at publish-time (canonical AUDIT_EVENTS IDs land in separate AUDIT_EVENTS amendment SI):
- `forms.publish.l3_dual_control_violation` (Cat B) — missing or invalid approval
- `forms.publish.l3_approval_artifact_drift` (Cat B) — approval/edit-log fingerprint mismatch at publish time
- `forms.publish.l3_live_state_unprovenanced` (Cat B) — live eligibility_logic leaf doesn't trace to an active approved edit
- `forms.l3_edit_log.superseded` (Cat B) — corrective supersession recorded
